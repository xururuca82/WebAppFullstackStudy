루비: 객체지향 언어

객체지향이란? - 절차지향(순서)

절차지향의 단점 : 협업 어려움, 유지보수 어려움, 가독성 어려움

다익스트라 - 구조적 프로그래밍 - 객체지향

상향식(bottom top) - 부품을 만들어 놓고 필요에 따라 사용

객체 지향 :
   1.클래스(class)-부품설계도(메타-공통관념들을 모아놓은 것)-메모리에 올라가지 않음
   2.인스턴스(instance)-실제부품(객체 Object)-메모리(현실세계)에 올라감

literal 형태(순수 숫자, 순수 문자)도 class로 인식함. 함수 바로 사용가능.(5.time() 이런식으로)

객체 지향의 속성:
   1.캡슐화 : 공통 분모를 모아서 분류 - class를 만듬
   2.은닉화 : 내부를 굳이 공개하지 않는다 - 마음대로 못 고치도록
   3.상속 : 기존 캡슐화된 속성을 상속받고 내가 원하는 속성을 추가로 부여한다. - 함수(메소드)를 다시 구현한다(Overiding)
   4.다형성 : 메소드에 인자값을 추가하거나, 다른 형태를 구현하도록 한다. - Overloading
